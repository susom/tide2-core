# Security Policy

## Supported Versions

We release patches for security vulnerabilities for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

If you discover a security vulnerability in TIDE 2.0, please use GitHub's private
vulnerability reporting feature by clicking **"Report a vulnerability"** on the
[Security tab](https://github.com/susom/tide2-core/security/advisories/new) of
this repository. This ensures the report is handled confidentially.

Please include the following information in your report:

- Type of issue (e.g., buffer overflow, SQL injection, cross-site scripting)
- Full paths of source file(s) related to the issue
- The location of the affected source code (tag/branch/commit or direct URL)
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit it

We will acknowledge receipt of your vulnerability report within 48 hours, and we will
send a more detailed response indicating the next steps in handling your submission.
After the initial reply, we will keep you informed of the progress being made towards
a fix and full announcement, and may ask for additional information or guidance.

## Preferred Languages

We prefer all communications to be in English.
